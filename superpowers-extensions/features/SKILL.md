# Product Features Knowledge Base

Use this with the superpowers plugin to keep `.context/features` aligned when project capabilities evolve.

## Purpose

- Maintain a business-oriented map of what the project does.
- Keep feature-level context useful for AI agent retrieval and decision-making.
- Avoid implementation details; this is a product/behavior layer.

## What belongs here

- User-facing capabilities and workflows.
- Business rules, constraints, permissions, and outcomes.
- High-level feature boundaries and relationships.

## What does not belong

- Model names, endpoints, migrations, or schema details.
- Implementation plans or testing checklists.
- Temporary brainstorming notes.

## Update rule

- Update feature docs when user-facing behavior changes.
