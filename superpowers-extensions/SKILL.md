# Manage Project Context

Use this with the superpowers plugin to standardize where project context is stored.

## STRUCTURE

```text
./.context
├── features/    # Knowledge base for what the project does
├── specs/       # Brainstormed work managed by superpowers/brainstorming
└── specs-plans/  # Active implementation plans managed by superpowers/writing-plans
```
